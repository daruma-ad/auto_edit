# AIエージェントと作る！完全自動の「プロ仕様動画編集ツール」開発記

動画編集において、最も時間がかかり、かつ単調な作業。それは「無音部分のカット（ジェットカット）」と「テロップ（字幕）入れ」ではないでしょうか。

今回は、AIエージェントとペアプログラミングを行い、**動画を投げ込むだけで「無音カット・自然な改行のテロップ・動画レンダリング・Premiere Pro連携」までを全自動で行ってくれるPythonスクリプト**を構築した過程をご紹介します。

---

## 💡 目指したゴール

単なる自動化ではなく、**「プロの仕事に使えるレベル」** を目指しました。
具体的には以下のような要件です。

1. **ジェットカットの完全自動化**: 適切な間（ま）を残しつつ、無音を自動で詰める。
2. **高精度な文字起こしと緻密な字幕ルール**:
   * Whisper (`large-v3`) を使用して単語レベルで時間を同期。
   * **1行最大30文字制限**。
   * 19文字を超える場合は、**日本語の文節を自然に区切って均等に2行にする**。
   * 文字数に応じてフォントサイズを自動調整（72px 〜 42px）。
3. **プロ用エディタとの連携**:
   * 自動編集の結果を **Adobe Premiere Pro** で再編集できるよう、`XML` と `SRT` 形式でも出力する。
4. **すべてをワンボタンで実行**: 
   * Python、FFmpeg、Remotion (Reactによる動画生成) を連携させ、1つのスクリプトで完結させる。

---

## 🛠️ 技術スタック

* **基本制御**: Python (`auto_edit_pro.py`)
* **無音検出 (VAD)**: Silero VAD (ONNX Runtime)
* **動画/音声処理**: FFmpeg
* **文字起こし**: OpenAI Whisper
* **日本語の自然な文節分割**: BudouX (Google)
* **動画レンダリング**: Remotion

---

## 🚀 こだわりポイントと実装の裏側

### 1. ジェットカットの長尺動画対応 (FFmpeg Chunked Concat)
無音検出には高速な「Silero VAD」を採用しました。しかし、ここで壁にぶつかりました。
検出したカット箇所があまりに多すぎると、FFmpeg に渡すコマンドラインの文字数が Windows の制限を超えてしまい、エラーになったのです。

**【解決策】**
カット情報を一度に渡すのではなく、「10セグメントずつ細かく動画をカットして保存し、最後にすべてをつなぎ合わせる（Chunked Concat）」という方式を実装することで、長時間の動画にも安定して対応できるようになりました。

### 2. BudouX による劇的な「読みやすさ」の向上
一般的な自動字幕生成では、「てにをは」など変な位置で改行されてしまい、結局手直しが必要になりがちです。

**【解決策】**
Googleが開発した軽量な分かち書きモデル **「BudouX」** を導入しました。
設定した「1行30文字」というルールの中で、18文字を超える場合は、BudouXを使って文節を解析し、ちょうど半分の文字数に近い「自然な区切り」で2行に分割するロジックをプログラミングしました。

### 3. 音声とテロップの「ミリ秒単位の同期ズレ」との戦い
終盤、大きな問題が発生しました。生成された動画を見ると、テロップが表示されるタイミングと、実際に音声が発音されるタイミングがズレていたのです。

**【原因と解決策】**
原因は、Whisperに「カット前の長い音声」を渡して解析していたことでした。
無音カットを行った後の動画映像に対し、カット前の音声のタイムスタンプを使っていたためズレが生じていました。
**「Step 3: カット済みの動画から、再度音声を抽出し直して Whisper に渡す」** という工程を追加することで、ミリ秒単位で完璧に同期するようになりました。

### 4. Premiere Pro への「タイムライン復元」書き出し
「AIの自動編集は便利だけど、結局最後は人間の目で見て、少し間を広げたり、テロップの色を変えたりしたい」という実務のニーズに応えるため、連携機能を実装しました。

**【解決策】**
自動カットの情報を「FCP7 XML形式」で書き出すスクリプトを組み込みました。Premiere Pro でこのXMLを読み込むと、カット済みのクリップが並んだタイムラインがそのまま復元されます。同時に書き出した `SRT` ファイルをキャプショントラックに放り込めば、AIの下書きからスムーズにプロの仕上げ作業へと移行できます。

## 📊 最終的な成果

テスト用動画（約3分半）で実行した結果がこちらです。

* **元動画の長さ**: 209.69 秒
* **カット後の長さ**: 96.32 秒
* **カット率**: **54.1%** （無音部分を半分以上カット！）
* **生成テロップ件数**: 49 件

ワンボタンを押してしばらく待つだけで、これだけの作業が自動で終わるようになりました。テロップも文節で綺麗に改行されており、Remotion による焼き付け動画 (`final.mp4`) も美しいUIを保っています。

## 🎯 おわりに

今回は、複数のツールやライブラリを組み合わせ、本番運用に耐えうる「プロ仕様の自動編集スクリプト」を構築しました。
パスの問題や同期ズレ、テンソルの形状エラーなど数々のトラブルがありましたが、AIと対話しながら柔軟にスクリプトを書き換えていくことで、見事なワークフローが完成しました。
