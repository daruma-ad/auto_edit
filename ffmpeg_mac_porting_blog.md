# AI動画編集スクリプトのMac移植とFFmpeg「魔の仕様」との戦い

Windows環境で稼働していたPython × Remotionベースの「AI自動動画編集（無音カット＆テロップ生成）スクリプト」を、Mac（Apple Silicon）環境へ移植しました。その際、単純なパスの修正だけでは済まず、FFmpegの深い仕様やコーデック周りで数々の罠にハマりました。

この記事では、発生した不具合とその根本的な解決策までの過程を記録します。

## 1. Mac環境への移行と基本設定・デッドロック対応
まずはOSの違いによるエラー潰しからスタートしました。
- `npx.cmd` を `npx` に変更し、絶対パスから相対パス運用へ修正。
- 処理速度を確保するため、Whisperモデルを `large-v3` からCPUでもサクサク動く `small` モデルへ変更。
- **デッドロック（フリーズ）の回避**: `subprocess.run` でFFmpegを実行する際、標準出力をキャプチャしたまま放置するとパイプのバッファが詰まってフリーズすることが発覚しました。検証系以外のコマンドは出力を `DEVNULL` に捨てるよう修正し、フリーズを根本から解決しました。

## 2. プレビューで動画・音声が再生されない
環境構築が完了し「いざ実行！」という段階で、Remotionのプレビュー（`npm run dev`）画面にテロップは表示されるものの、動画も音声も止まったまま動かない現象が発生しました。

原因は**最終出力ファイルの配置漏れ**でした。
Remotionが要求する音声ファイル（`voice_audio.wav`）や、最新の変換済み動画（`cut_video.mp4`）が `public` フォルダへ正しくコピーされていませんでした。スクリプトにファイルのコピー処理を確実に追加して解決しました。

## 3. 「動画がカットされない」「テロップの開始がズレる」
動画が動くようになり、ようやく完成かと思いきや **「テロップは進むのに、元の動画が全く無音カットされずにそのまま再生される」「最初のテロップが出るまで14秒もズレている」** という致命的な問題が発覚しました。

### 調査から分かった2つの原因
1. **VAD（音声区間検出）が役に立っていない**
   元のスクリプトでは `silero-vad` を用いて無音区間を判定しカットしていましたが、ノイズに敏感に反応しすぎた結果、動画のほぼ全域を「音声あり」と判定していました。
2. **HEVC動画に対する `select` フィルターの罠**
   VADを捨て、Whisperの文字起こし区間だけを使ってカットしようとしました。しかし、FFmpegの **`select` フィルターがHEVC（iPhone撮影の.mov）に対して実質的に機能しなかった** のです。
   いくら発話区間だけを `select` で時間指定しても、出力される動画の長さは元の209秒のまま全く短くなりませんでした。

## 4. 解決策：trim と concat による一撃結合へ大改修
`select` フィルターの挙動不審に頼るのをやめ、**各発話区間を `trim`（映像） / `atrim`（音声） フィルターで個別に切り出し、それを `concat` フィルターで1本の動画に結合する** という力技かつ確実な方式へスクリプトを全面リファクタリングしました。

```python
# 修正後のイメージ（Pythonコード抜粋）
filter_parts = []
for i, seg in enumerate(segments):
    # 各セグメントを個別にtrim
    filter_parts.append(f"[0:v]trim=start={seg['start']}:end={seg['end']},setpts=PTS-STARTPTS[v{i}]")
    filter_parts.append(f"[0:a]atrim=start={seg['start']}:end={seg['end']},asetpts=PTS-STARTPTS[a{i}]")

# concat で1本に再構築
concat_inputs = "".join(f"[v{i}][a{i}]" for i in range(n))
filter_parts.append(f"{concat_inputs}concat=n={n}:v=1:a=1[outv][outa]")
```

### タイムスタンプの「再マッピング」
動画が細切れを結合して新しく作られたため、**Whisperが持っている「元の動画のタイムスタンプ」と、完成した「無音カット済み動画」の時間軸が合わなくなります。**
そこで、テロップを生成する際に、古い時間軸から新しい時間軸へタイムスタンプを再計算（再マッピング）するロジックを独自に組み込みました。

## 結果とまとめ
これらの改修を施した結果、驚くほど完璧に動作するようになりました。

- **総時間**: 209秒の元動画が、無駄のない **141秒に圧縮** されました（68秒の無音カットに成功！）。
- **同期**: テロップはズレることなく **0.00秒から完璧にスタート** し、映像と音声に完全に同期しています。

### 学び
1. iPhoneで撮影した **HEVC(.mov)** などの動画をFFmpegで時間指定カットする際は `select` は避け、**`trim` + `concat`** を使うべし。
2. Whisperの文字起こしを元に動画全体を物理カットする場合、テロップをかぶせるなら**時間軸の再マッピング**処理が必須。
3. Pythonの `subprocess` によるFFmpeg実行時は、パイプバッファの詰まり（デッドロック）と引数のクォーテーション展開（`shell=True` の利用）に気をつける。

紆余曲折ありましたが、Mac上でも「動画を置いて実行するだけ」で無音カット＆テロップ自動生成が行える快適な環境が完成しました！
